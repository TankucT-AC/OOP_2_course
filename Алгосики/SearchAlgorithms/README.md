# Лабораторная работа по алгоритмам поиска

## Цель работы
Сравнение эффективности различных алгоритмов поиска в массивах с анализом временной сложности в нотациях Θ (тета), O (омикрон) и Ω (омега):
- Better Linear Search (BLS)
- Sentinel Linear Search (SLS) 
- Ordered Array Search (OAS)
- Binary Search (BS)

## Описание алгоритмов

### Better Linear Search (BLS)
- **Тип**: Линейный поиск
- **Особенности**: Поиск с оптимизацией - выход при первом найденном элементе
- **Сложность**:
  - Θ(n) - средний случай
  - O(n) - худший случай (элемент в конце или отсутствует)
  - Ω(1) - лучший случай (элемент первый)

### Sentinel Linear Search (SLS)
- **Тип**: Линейный поиск с барьером
- **Особенности**: Использует временную замену последнего элемента для исключения проверки границ
- **Сложность**:
  - Θ(n) - средний случай
  - O(n) - худший случай (элемент предпоследний или отсутствует)
  - Ω(1) - лучший случай (элемент первый)

### Ordered Array Search (OAS)
- **Тип**: Линейный поиск в отсортированном массиве
- **Особенности**: Выход при достижении элемента больше или равного искомому
- **Сложность**:
  - Θ(n) - средний случай
  - O(n) - худший случай (элемент последний или больше всех)
  - Ω(1) - лучший случай (элемент первый)

### Binary Search (BS)
- **Тип**: Двоичный поиск
- **Особенности**: Работает только на отсортированных массивах, использует стратегию "разделяй и властвуй"
- **Сложность**:
  - Θ(log n) - средний случай
  - O(log n) - худший случай (элемент отсутствует или на границе)
  - Ω(1) - лучший случай (элемент в середине)

## Сравнительная таблица сложности

| Алгоритм | Лучший случай Ω | Средний случай Θ | Худший случай O | Память |
|----------|-----------------|------------------|-----------------|--------|
| BLS      | Ω(1)            | Θ(n)             | O(n)            | O(1)   |
| SLS      | Ω(1)            | Θ(n)             | O(n)            | O(1)   |
| OAS      | Ω(1)            | Θ(n)             | O(n)            | O(1)   |
| BS       | Ω(1)            | Θ(log n)         | O(log n)        | O(1)   |

## Параметры тестирования

- **Диапазон значений**: от -1000 до 1000
- **Размеры массивов**: 10M, 25M, 30M, 50M, 75M, 100M элементов
- **Типы поиска**:
  1. **Лучший случай** - первый элемент (для линейных), средний элемент (для бинарного)
  2. **Средний случай** - элемент в середине массива
  3. **Худший случай** - отсутствующий элемент

## Метрики сравнения

- **Время выполнения** (в наносекундах)
- **Counter1**: Количество сравнений элементов (отражает O-сложность)
- **Counter2**: Количество итераций циклов (отражает Ω-сложность)

## Теоретический анализ

### Better Linear Search (BLS)
- **Ω(1)**: Элемент находится на первой позиции
- **Θ(n)**: Элемент в середине массива (средний случай)
- **O(n)**: Элемент в конце или отсутствует

### Sentinel Linear Search (SLS)  
- **Ω(1)**: Элемент находится на первой позиции
- **Θ(n)**: Устраняет одну проверку на итерацию, но сложность та же
- **O(n)**: Элемент предпоследний или отсутствует

### Ordered Array Search (OAS)
- **Ω(1)**: Искомый элемент первый
- **Θ(n)**: В среднем поиск останавливается на середине
- **O(n)**: Элемент больше всех существующих

### Binary Search (BS)
- **Ω(1)**: Элемент точно в середине массива
- **Θ(log n)**: Стандартный случай деления пополам
- **O(log n)**: Гарантированная логарифмическая сложность
